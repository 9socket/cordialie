What is Cordialie?

Cordialie is a .py script to automate password profiling. It's highly dynamic and customizable, while being efficent and fast.

It's mainly more efficent towards specific targets/websites/companies, and relies on OSINT and social engineering.

It takes advantage of passwords with personal information, like "JohnSmith31" for example.

Even though this script does not directly launch any dictionary attacks, its purpose is to build up wordlist for penetration testing.

Hopefully this goes to show as a big example why personal information shouldn't be used in any password, and to practice basic saftey online.

Cordialie Usage

As mentioned earlier, Cordialie uses Argparse to set in its commands. There is a required set of arguements, then there are optional ones.

required:

Cordialie has a simple usage. It only takes 3 arguements to run. "cordialie.py -f (filename) -a (list amount) -k (keyword amount)".

Lets take a look at what each arguement does.

First, the file is defined. This is where the list gets printed to, and will most likley be in the form of a .txt file.

Next, you specify the list amount. This defines how many words you want generated on your list. So if you do "-a 1000", you'll get a thousand words.

If you do a large amount, like over a million for example, it may take a little longer then usual, but it'll work out fine.

Last, we need to tell it the keyword amount. This is how many keywords you want to specify for the generated list. This is where OSINT and social engineering comes in.

Thats it! All you need to run it, are those 3 simple arguements. So in the end it may look something like, "cordialie.py -f target.txt -a 100000 -k 10".

optional:

Cordialie comes with optional arguements as well, to make the script more suited for different needs. Lets take a look and see what they are.

• -pm : This is [print_mode]. When this is specified, it'll show all the combinations being generated on your commandline/terminal.
